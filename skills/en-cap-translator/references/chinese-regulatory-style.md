# 中文法规文体指南

当目标交付物面向工程、法规、测试或合规团队时，中文应采用“法规/协议/标准”文体，而不是逐词直译的说明文口吻。

## 总体原则

- 以正式、克制、可审阅为目标，不追求营销式流畅。
- 优先保证义务强度、条件边界、计分逻辑和定义关系准确。
- 中文句子可适度重组，但不得改变约束关系、时序关系或条件范围。
- 同一文档内，核心术语和句式必须稳定，不要同义改写过多。

## 义务与判断句型

- `shall` -> `应`
- `must` -> `必须`
- `may` -> `可`
- `is encouraged to` -> `鼓励`
- `is forbidden from` -> `不得`
- `is allowed to` -> `允许`
- `is considered` / `is deemed` -> `视为`
- `is classified as` -> `认定为`
- `is notified / informed` -> `应告知` / `应提示`

## 计分与判定句型

优先使用以下表达：

- `is eligible to score points` -> `可计分`
- `is awarded` -> `计分` / `可计分`
- `to be awarded a PASS` -> `方可判定为通过`
- `meets the performance criteria` -> `满足性能标准`

避免以下直译腔：

- `具有得分资格`
- `可获得该项评分`
- `被授予 PASS`

## 推荐替换

| 不推荐 | 推荐 |
| --- | --- |
| 不具功能 | 失效 |
| 有力信息 | 充分说明 |
| 覆盖误检情况 | 作为误检处理机制 |
| 不被允许参加 | 不再获准参加 |
| 判定受影响要求的评估及后续结果 | 确定受影响要求的评估及后续结果 |
| 一种能够判定……的系统 | 用于判断……的系统 |

## 定义句写法

定义句优先使用以下结构：

- `X - 指……`
- `X - 系指……`
- `X - 用于……`

示例：

```text
Driver State Monitoring (DSM) - 用于判断驾驶员状态的系统。
```

比起下面这种更适合协议文体：

```text
Driver State Monitoring (DSM) - 一种能够判定驾驶员状态的系统。
```

## 专有名词与项目名

- 对官方项目名、协议名、评分项名，首次出现时优先保留英文原名。
- 如中文表达有助于团队阅读，可在中文后补英文括号。

示例：

- `Crash Avoidance（碰撞避免）`
- `Driver Monitoring`
- `Emergency Function (EF)`

## 当前文档中的典型修正

- `具有得分资格` -> `可计分`
- `不具功能` -> `失效`
- `有力信息` -> `充分说明`
- `被判定为分心` -> `被认定为分心`
- `可判定为无响应驾驶员` -> `可认定驾驶员处于无响应状态`
